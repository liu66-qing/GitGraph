## 实现要点

### 路径图区域(最关键)

这是与之前实现差异最大的部分。原型图中路径图是**从左到右水平蜿蜒**的,不是从下到上。

实现方式:
1. 路径图容器: `width: 100%; height: clamp(320px, 40vh, 400px); position: relative; overflow: hidden;`
2. 背景: 一张像素风场景图(用已有资源拼接,或用 AI 生成,或用占位)覆盖整个容器。
3. 4 个节点卡片: `position: absolute`,沿水平方向从左到右分布,垂直方向略有起伏(模拟蜿蜒)。
4. 节点之间用 SVG 绿色箭头连接。
5. 整个容器有像素风浅蓝色边框。

节点定位参考(百分比):
```
节点1: left: 5%,  top: 45%
节点2: left: 28%, top: 35%
节点3: left: 52%, top: 45%
节点4: left: 76%, top: 35%
```

### 侧边栏

- 使用 Mantine AppShell.Navbar,宽度 `240px`。
- 导航项用自定义组件,不要用默认 NavLink 样式。
- 每项: 像素图标 + 文字 + 右侧数字徽章。
- 底部用户区: 像素头像 + 名字 + XP 进度条。

### 底部三栏

- 使用 CSS Grid: `grid-template-columns: 1fr 1fr 1fr; gap: 20px;`
- 每个卡片: 白色背景,圆角 `12px`,浅灰边框,内部 padding `20px`。
- 在 `< 1280px` 时改为 `grid-template-columns: 1fr;`(堆叠)。

### 背景场景图

**单张融合背景图:**

文件: `frontend/src/assets/pixel/backgrounds/home-hero-combined.png`

这张图融合了天空(Hero区)和村庄场景(路径图区),是一张完整的背景。图片结构:
- **上部约55%:** 纯净蓝天 + 白云 + 远处绿色山脉。标题、副标题、输入框放在这个区域。
- **下部约45%:** 村庄场景(小屋、风车、桥、小溪、宝箱、旗帜),已自带深色圆角边框。路径图节点卡片放在这个区域。

**CSS 实现:**
```css
/* 上半区:一张图搞定 Hero + 路径图 */
.homepage-upper-section {
  background-image: url('/src/assets/pixel/backgrounds/home-hero-combined.png');
  background-size: 100% auto;
  background-position: top center;
  background-repeat: no-repeat;
  image-rendering: pixelated;
  position: relative;
  min-height: clamp(620px, 78vh, 780px);
  padding: 0 clamp(20px, 3vw, 40px);
}

/* 路径图容器:透明!不需要自己的背景和边框 */
/* 背景图本身已经画了村庄场景和边框 */
.journey-map-container {
  position: relative;
  height: clamp(260px, 33vh, 340px);
  /* 无 background-image, 无 border */
}

/* 下半区:三栏卡片 */
.homepage-lower-section {
  background: #f8f9fa;
  padding: 24px clamp(20px, 3vw, 40px);
}
```

**内容定位要点(配合背景图):**

Hero 区内容(标题+输入框)必须落在背景图的天空区域(上部55%):
- 标题区整体 `padding-top: clamp(40px, 6vh, 80px)`
- 标题、副标题、输入框、热门标签紧凑排列,总高度控制在 `250-300px` 内
- 不要让输入框或热门标签掉到村庄场景区域里

路径图节点必须落在背景图的村庄区域(下部45%):
- 4个节点卡片水平均匀分布
- 垂直方向居中于村庄场景区域
- 节点位置:
  - ① left: 6%,  top: 50% (相对于路径图容器)
  - ② left: 28%, top: 40%
  - ③ left: 53%, top: 50%
  - ④ left: 76%, top: 40%

**关键:** 路径图容器不设 background-image 也不设 border,因为背景图自带了村庄场景和深色圆角边框。容器只是一个透明的定位层。

**底部三栏卡片** 在背景图下方,用独立的浅灰/白色背景,与像素场景完全分离。

## 强制约束(反复出现的问题,必须遵守)

### 约束1: 所有内容必须在第一屏内完整可见

在 1080p 分辨率(1920×1080 或 1440×900)下,首页所有内容(标题+输入框+路径图+底部三栏卡片)必须在不滚动的情况下完整可见。

实现方式:
- 上半区(Hero + 路径图)总高度不超过 `65vh`
- Hero 区(标题+输入框+标签)紧凑排列,总高度控制在 `220-260px`
- 路径图容器高度: `clamp(240px, 30vh, 300px)`,不要更高
- 底部三栏卡片高度约 `200-240px`
- 各区域间距用 `12-16px`,不要用 `24px` 以上的大间距

如果内容被挤出屏幕,优先缩小:路径图高度 > Hero 区 padding > 底部卡片高度。

### 约束2: 路径图节点必须是像素风卡片

节点卡片不能是普通的白色 div + 文字。必须有像素游戏感。

节点卡片样式要求:
```css
.journey-node-card {
  background: #fffef5;  /* 羊皮纸/米白色 */
  border: 3px solid #5c4a3a;  /* 像素风深棕色边框 */
  border-radius: 0;  /* 像素风不要圆角!用直角 */
  box-shadow: 
    4px 4px 0 #3a2d1f,   /* 右下像素阴影 */
    -1px -1px 0 #d4c4a0;  /* 左上高光 */
  padding: 12px 16px;
  image-rendering: pixelated;
  font-family: inherit;  /* 正文字体,不用像素字体 */
}

.journey-node-number {
  width: 28px;
  height: 28px;
  background: #22c55e;
  border: 2px solid #1a8c4a;
  border-radius: 0;  /* 像素风方形,不是圆形 */
  color: white;
  font-weight: bold;
  display: flex;
  align-items: center;
  justify-content: center;
  box-shadow: 2px 2px 0 #145a30;
}
```

如果项目中有 Quintino Pixel UI 的木质面板资源(`frontend/src/assets/pixel/ui/`),优先使用 9-slice 木质面板作为节点卡片背景。

### 约束3: 路径图第一个节点不能被裁切

4个节点必须全部完整可见,不能有任何一个被容器边缘裁切。

实现方式:
- 路径图容器: `overflow: visible`(不是 hidden!)
- 第一个节点 left 值不小于 `6%`(给左边留出卡片宽度的空间)
- 最后一个节点 right 值不小于 `4%`
- 或者用 `padding: 0 20px` 给容器内部留出安全边距

### 约束4: 标题装饰图标必须是像素风

标题上方的小装饰图标(植物/树苗)必须使用像素风资源,不能是:
- 普通的圆形 div
- SVG 圆形图标
- Lucide/FontAwesome 图标

使用方式(按优先级):
1. 从 `frontend/src/assets/pixel/` 中找一个合适的像素 sprite(树苗、水晶、星星等)
2. 用 CSS `image-rendering: pixelated` + 小尺寸 PNG
3. 如果实在没有合适资源,用一个 `32×32` 的像素风 CSS 方块组合(不是圆形)

### 约束5: 节点之间的箭头也要像素风

节点间的连接箭头不能是普通的 SVG 线条或 CSS border。应该是:
- 像素风的绿色箭头图标(从资源包中取,或用 CSS box-shadow 模拟像素箭头)
- 或者用像素风的 `>>>` 符号
- 箭头颜色: `#22c55e`(绿色)

## 禁止事项

- 不要把路径图做成垂直的(从下到上)。原型图明确是水平从左到右。
- 不要用 Mantine Timeline 组件。
- 不要用纯文字列表代替路径图。
- 不要让侧边栏遮挡主内容。
- 不要给主内容区设置固定宽度(用 fluid)。
- 不要用 NES.css。
- 不要手搓像素角色(用已有 sprite 或占位图)。
- 不要给路径图容器设 `overflow: hidden`(会裁切边缘节点)。
- 不要用圆角(`border-radius`)做像素风卡片——像素风用直角。
- 不要用普通圆形 div 做装饰图标——用像素资源。
- 不要让任何内容超出首屏(1080p 下不滚动即可看到全部)。
- 参考 PRD 第17节的所有禁止 CSS 模式。
